## Overview

The `ProcessedImagesList` component displays a responsive grid of processed images with their metadata, including file size, optimization statistics, and detected format. It handles loading states and empty states gracefully.

## Features

- Responsive grid layout (1-4 columns based on screen size)
- Image previews with optimized display
- File size and format information
- Optimization statistics (reduction percentage and size)
- Loading state with spinner
- Empty state placeholder
- File name tooltips for truncated names
- Automatic format detection from MIME types

## Display states

The component handles three distinct states:

### Loading state

When `loading={true}`:

```jsx
<ProcessedImagesList images={[]} loading={true} />
```

Displays a centered spinner with "Procesando imágenes..." text.

### Empty state

When `images` is empty and not loading:

```jsx
<ProcessedImagesList images={[]} loading={false} />
```

Displays an empty state placeholder with "No hay imágenes procesadas" message.

### Grid state

When images are available:

```jsx
<ProcessedImagesList images={processedImages} loading={false} />
```

Displays a responsive grid of image cards.

## Format detection

The component includes intelligent format detection:

```jsx
const detectRealFormat = (image) => {
  // First, check processedFile MIME type
  if (image.processedFile && image.processedFile.type) {
    const mimeType = image.processedFile.type.toLowerCase();
    if (mimeType.includes('jpeg') || mimeType.includes('jpg')) return 'JPG';
    if (mimeType.includes('png')) return 'PNG';
    if (mimeType.includes('webp')) return 'WEBP';
    if (mimeType.includes('gif')) return 'GIF';
  }
  
  // Fallback to file extension
  const ext = getFileExtension(image.newName || image.name);
  return ext.toUpperCase();
};
```

<Note>
  Format detection prioritizes MIME type from the processed file over filename extension for accuracy.
</Note>

```jsx

    ↓ {image.optimization.reductionPercent}% ({formatFileSize(image.optimization.reduction)})
  </Text>

```

**Example output:** `↓ 25% (524 KB)`

## Data validation

The component filters out invalid images before rendering:

```jsx
const validImages = images.filter((img) => img && img.id);
```

<Warning>
  Images without an `id` property are silently filtered out to prevent rendering errors.
</Warning>

## Preview image selection

The component uses a fallback strategy for preview images:

```jsx
src={image.processedPreview || image.preview}
alt={image.newName || image.name}
```

1. **processedPreview** - Optimized preview after processing (preferred)
2. **preview** - Original preview (fallback)
