## Overview

The `useImageRename` hook provides reactive, automatic renaming of processed images with support for sequential numbering, text addition/replacement, and filename sanitization. Renaming happens automatically via `useEffect` whenever processed images or rename settings change.

## Import

```javascript
import useImageRename from '@/hooks/useImageRename';
```

## Parameters

<ParamField path="processedImages" type="array" required>
  Array of processed image objects from `useImageProcessor`. The hook monitors this array and automatically renames images when it changes.
</ParamField>

## Usage

```javascript
function App() {
  const { processedImages } = useImageProcessor(uploadedImages);
  
  const {
    renamedImages,
    renameSettings,
    toggleCleanText,
    changeRenameMode,
    updateSequentialSettings,
    updateAddTextSettings,
    updateReplaceTextSettings,
  } = useImageRename(processedImages);

  return (
    <RenamePanel
      settings={renameSettings}
      onToggleClean={toggleCleanText}
      onModeChange={changeRenameMode}
      onSequentialUpdate={updateSequentialSettings}
      onAddTextUpdate={updateAddTextSettings}
      onReplaceTextUpdate={updateReplaceTextSettings}
      preview={renamedImages}
    />
  );
}
```

## Rename Modes

### Sequential Mode

Generates numbered filenames with customizable prefix, start number, and zero-padding:

```javascript
// Default settings
{ prefix: 'IMG_', startNumber: 1, digits: 3 }
// Results: IMG_001.jpg, IMG_002.jpg, IMG_003.jpg

// Custom settings
{ prefix: 'vacation_', startNumber: 50, digits: 4 }
// Results: vacation_0050.jpg, vacation_0051.jpg, vacation_0052.jpg
```

Source: `src/hooks/useImageRename.js:54-59`

### Add Text Mode

Adds custom text as prefix or suffix to existing filenames:

```javascript
// Prefix mode
{ position: 'prefix', text: 'optimized_' }
// photo.jpg -> optimized_photo.jpg

// Suffix mode
{ position: 'suffix', text: '_web' }
// photo.jpg -> photo_web.jpg
```

Source: `src/hooks/useImageRename.js:62-67`

### Replace Text Mode

Replaces text patterns using regular expressions:

```javascript
// Simple replacement
{ search: 'IMG', replace: 'Photo' }
// IMG_1234.jpg -> Photo_1234.jpg

// Regex pattern (global replacement)
{ search: '\\s+', replace: '_' }
// my photo 001.jpg -> my_photo_001.jpg
```

Source: `src/hooks/useImageRename.js:70-75`

## Filename Sanitization

When `cleanText` is enabled (default), filenames are automatically sanitized:

```javascript
const cleanFileName = (name) => {
  return name
    .replace(/[^\w\s.-]/g, '')  // Remove special characters
    .replace(/\s+/g, '_')        // Replace spaces with underscores
    .toLowerCase();              // Convert to lowercase
};
```

**Examples:**

- `My Photo (1).jpg` → `my_photo_1.jpg`
- `IMG@2024#01!.png` → `img202401.png`
- `Vacation - Beach.jpeg` → `vacation_-_beach.jpeg`

Source: `src/hooks/useImageRename.js:43-51`

## Extension Handling

The hook intelligently determines the correct file extension:

<Steps>
  <Step title="Check MIME Type">
    If the processed file has a MIME type, extract extension from it
    ```javascript
    'image/jpeg' -> 'jpg'
    'image/png' -> 'png'
    'image/webp' -> 'webp'
    ```
  </Step>
  
  <Step title="Fallback to Filename">
    If no MIME type, extract extension from the filename
    ```javascript
    'photo.jpg' -> 'jpg'
    'image.PNG' -> 'PNG'
    ```
  </Step>
  
  <Step title="Apply to Renamed File">
    The extension is preserved and applied to the new name
    ```javascript
    baseName = 'IMG_001'
    extension = 'jpg'
    newName = 'IMG_001.jpg'
    ```
  </Step>
</Steps>

## Reactive Updates

The hook uses `useEffect` to automatically rename images when dependencies change:

```javascript
useEffect(() => {
  // Rename all images whenever:
  // 1. processedImages array changes
  // 2. renameSettings object changes
  const renamed = processedImages.map((img, index) => {
    // Apply rename logic...
    return { ...img, renamed: true, newName };
  });
  
  setRenamedImages(renamed);
}, [processedImages, renameSettings]);
```

**This means:**
- Settings changes instantly update all filenames
- New processed images are automatically renamed
- No manual trigger required

## Edge Cases

### Empty processedImages

When `processedImages` is empty or null, `renamedImages` is automatically cleared:

```javascript
if (!processedImages || processedImages.length === 0) {
  setRenamedImages([]);
  return;
}
```

Source: `src/hooks/useImageRename.js:35-38`

### Empty Search Text

In replace mode, if search text is empty, the original name is preserved:

```javascript
const replaceText = (name) => {
  const { search, replace } = renameSettings.replaceText;
  if (!search) return name; // No replacement
  return name.replace(new RegExp(search, 'g'), replace);
};
```

Source: `src/hooks/useImageRename.js:70-75`

### Empty Add Text

In add text mode, if text is empty, the original name is preserved:

```javascript
const addText = (name) => {
  const { position, text } = renameSettings.addText;
  if (!text) return name; // No addition
  return position === 'prefix' ? `${text}${name}` : `${name}${text}`;
};
```

Source: `src/hooks/useImageRename.js:62-67`

## Data Flow

The hook is the final step in the PixelFlow processing pipeline:

```
useImageUpload (upload)
    ↓
useImageProcessor (process) 
    ↓
useImageRename (rename) ← You are here
    ↓
ProcessedImagesList + DownloadButton
```

Rename happens **reactively** (via useEffect), while processing happens **manually** (via button click).
