## Overview

The `useImageProcessor` hook manages the complete image processing pipeline in PixelFlow. It handles optimization, resizing, cropping, and format conversion with support for batch processing and aspect ratio preservation. Processing is triggered manually via the `processImages` function.

## Import

```javascript
import useImageProcessor from '@/hooks/useImageProcessor';
```

## Parameters

<ParamField path="uploadedImages" type="array" required>
  Array of uploaded image objects from `useImageUpload`. Each object should contain `file`, `name`, `size`, and other metadata.
</ParamField>

## Usage

```javascript
function App() {
  const { uploadedImages } = useImageUpload();
  
  const {
    processedImages,
    processingSettings,
    processing,
    processImages,
    toggleOptimize,
    changeFormat,
    updateResizeSettings,
    updateCropSettings,
  } = useImageProcessor(uploadedImages);

  return (
    <>
      <ToolsPanel
        settings={processingSettings}
        onToggleOptimize={toggleOptimize}
        onFormatChange={changeFormat}
        onResizeUpdate={updateResizeSettings}
        onCropUpdate={updateCropSettings}
      />
      <button onClick={processImages} disabled={processing}>
        {processing ? 'Processing...' : 'Process Images'}
      </button>
    </>
  );
}
```

## Processing Pipeline

Operations are applied in the following order:

<Steps>
  <Step title="Optimization">
    Compresses image using `browser-image-compression`
    
    - Max size: 1MB (OPTIMIZE_MAX_SIZE_MB)
    - Max dimension: 1920px (OPTIMIZE_MAX_DIMENSION)
    - Web worker enabled for performance
  </Step>
  
  <Step title="Resize">
    Resizes image using `react-image-file-resizer`
    
    - Supports pixel (px) and percentage (%) units
    - Automatic aspect ratio preservation when enabled
    - Percentage is calculated from original dimensions
  </Step>
  
  <Step title="Crop">
    Crops image using custom canvas implementation
    
    - Applied after resize for accurate dimensions
    - Supports multiple position options
    - Custom width/height in pixels
  </Step>
  
  <Step title="Format Conversion">
    Converts image format using canvas `toBlob()`
    
    - JPG, PNG, WEBP, GIF support
    - Automatic MIME type detection
    - Extension updated in filename
  </Step>
</Steps>

```javascript
// Processing pipeline source code reference
// 1. Optimization (src/hooks/useImageProcessor.js:119-127)
if (processingSettings.optimize) {
  processedFile = await optimizeImage(processedFile, {
    maxSizeMB: OPTIMIZE_MAX_SIZE_MB,
    maxWidthOrHeight: OPTIMIZE_MAX_DIMENSION,
    useWebWorker: true,
  });
}

// 2. Resize (src/hooks/useImageProcessor.js:130-141)
if (processingSettings.resize.enabled) {
  processedFile = await resizeImage(
    processedFile,
    resizeDimensions.width,
    resizeDimensions.height,
    processingSettings.resize.maintainAspectRatio
  );
}

// 3. Crop (src/hooks/useImageProcessor.js:164-202)
if (processingSettings.crop.enabled) {
  processedFile = await cropImage(
    processedFile,
    cropWidth,
    cropHeight,
    processingSettings.crop.position || 'center'
  );
}

// 4. Format Conversion (src/hooks/useImageProcessor.js:205-224)
if (processingSettings.format !== 'original') {
  processedFile = await convertImageFormat(
    processedFile,
    processingSettings.format
  );
}
```

## Batch Processing

Images are processed in batches for optimal performance:

- **Batch Size**: 5 images at a time (PROCESSING_BATCH_SIZE)
- **Parallel Processing**: Images within a batch process simultaneously
- **Sequential Batches**: Batches process one after another
- **Error Isolation**: Failed images don't stop batch processing

## Aspect Ratio Preservation

When `maintainAspectRatio` is enabled, the hook intelligently adjusts dimensions:

```javascript
// Updating width auto-calculates height
updateResizeSettings({ width: 1200 });
// If aspect ratio is 4:3, height becomes 900

// Updating height auto-calculates width
updateResizeSettings({ height: 600 });
// If aspect ratio is 4:3, width becomes 800

// Toggling maintainAspectRatio captures current ratio
updateResizeSettings({ maintainAspectRatio: true });
// Current ratio: width / height is saved
```

## Percentage-Based Resizing

The hook supports percentage-based dimensions:

```javascript
// Resize to 50% of original dimensions
updateResizeSettings({ 
  enabled: true,
  unit: '%', 
  width: 50 
});

// Hook calculates actual pixel dimensions:
// Original 2000x1500 -> New 1000x750
```

Source: `src/hooks/useImageProcessor.js:95-114`

## Auto-Cleanup

The hook automatically clears processed images when:

- `uploadedImages` becomes empty
- `uploadedImages` changes (new upload replaces old)

Source: `src/hooks/useImageProcessor.js:306-311`

## Error Handling

Robust error handling ensures stability:

- Individual image errors are caught and logged
- Failed images return unprocessed with error message
- Format conversion errors fallback to original format
- Dimension calculation errors are logged and skipped

```javascript
try {
  // Process image operations...
  return processedImageObject;
} catch (error) {
  logger.error(`Error processing image ${img.name}:`, error);
  return {
    ...img,
    processed: true,
    processedPreview: img.preview,
    processedFile: img.file,
    error: error.message,
  };
}
```
