## Overview

The `useImageUpload` hook manages the complete image upload workflow in PixelFlow, including file validation, batch processing, memory optimization for large files, and preview generation. It processes images in batches to prevent memory issues and creates low-resolution previews for files larger than 10MB.

## Import

```javascript
import useImageUpload from '@/hooks/useImageUpload';
```

## Usage

```javascript
function App() {
  const { 
    uploadedImages, 
    loading, 
    handleUpload, 
    removeImage, 
    clearImages 
  } = useImageUpload();

  return (
    <UploadArea
      onUpload={handleUpload}
      loading={loading}
      uploadedImages={uploadedImages}
      onRemove={removeImage}
    />
  );
}
```

## Implementation Details

### Batch Processing

Images are processed in batches to optimize memory usage:

- **Batch Size**: 3 images at a time (defined by `UPLOAD_BATCH_SIZE`)
- **Progress Threshold**: Shows progress messages when uploading more than 5 images (`UPLOAD_PROGRESS_THRESHOLD`)
- **Processing Delay**: 10ms delay between batches to allow browser updates

### Memory Optimization

The hook implements intelligent memory management:

<CodeGroup>
```javascript Large Image Detection
// Files > 10MB are flagged as large images
const isLargeImage = file.size > LARGE_IMAGE_THRESHOLD; // 10MB

if (isLargeImage) {
  // Create low-resolution preview to save memory
  preview = await createLowResPreview(file);
} else {
  // Use standard preview for normal-sized images
  preview = getImagePreview(file);
}
```

```javascript Memory Warnings
// Estimate memory usage and warn if exceeds threshold
const estimatedMemory = estimateImageMemoryUsage(file);
const memoryWarning = estimatedMemory > MEMORY_WARNING_THRESHOLD;

if (memoryWarning) {
  logger.warn(`Large image detected: ${file.name}. May affect performance.`);
}
```
</CodeGroup>

### Resource Cleanup

The hook properly manages browser resources:

- **Single Image Removal**: Revokes blob URL when removing an image
- **Bulk Removal**: Revokes all blob URLs when clearing images
- **Large Images**: Skips URL revocation for low-res previews (different lifecycle)

```javascript
// Example: Resource cleanup on removal
const removeImage = (id) => {
  const imageToRemove = uploadedImages.find((img) => img.id === id);
  
  if (imageToRemove && imageToRemove.preview && !imageToRemove.isLargeImage) {
    URL.revokeObjectURL(imageToRemove.preview);
  }
  
  setUploadedImages((prev) => prev.filter((img) => img.id !== id));
};
```

## Constants Reference

The hook uses the following constants from `@/lib/constants`:

<ParamField path="UPLOAD_BATCH_SIZE" type="number" default="3">
  Number of images to process simultaneously
</ParamField>

<ParamField path="UPLOAD_PROGRESS_THRESHOLD" type="number" default="5">
  Minimum number of images to show progress messages
</ParamField>

<ParamField path="LARGE_IMAGE_THRESHOLD" type="number" default="10485760">
  File size in bytes (10MB) to trigger low-res preview mode
</ParamField>

<ParamField path="MEMORY_WARNING_THRESHOLD" type="number">
  Memory usage threshold for performance warnings
</ParamField>

## Error Handling

The hook implements comprehensive error handling:

- Failed image uploads return `null` and are filtered from results
- Individual image errors are logged but don't stop batch processing
- Success message shows actual count of successfully loaded images
- Preview URL errors during cleanup are caught and logged

```javascript
try {
  // Process image...
  return imageObject;
} catch (error) {
  logger.error(`Error processing image ${file.name}:`, error);
  return null; // Filtered out later
}
```
