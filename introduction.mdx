## What is PixelFlow?

PixelFlow is a client-side React 19 single-page application built for batch image processing. It enables you to optimize, resize, crop, convert formats, and rename multiple images simultaneously — all without sending your files to any server.

## Architecture Overview

PixelFlow follows a clean, modular architecture:

```
Processing Pipeline:
Upload → Optimize → Resize → Crop → Format Convert → Rename → Download
```

<Steps>
  <Step title="State Management">
    Three custom React hooks manage the entire application flow:
    - `useImageUpload` — handles file upload with batch loading (3 at a time)
    - `useImageProcessor` — applies transformations in a strict order
    - `useImageRename` — reactive renaming with three modes (sequential, add text, replace text)
  </Step>

  <Step title="Data Flow">
    ```
    uploadedImages → [manual trigger] → processedImages → [reactive] → renamedImages
    ```
    Processing is triggered manually by the user, while renaming happens reactively via `useEffect`.
  </Step>

  <Step title="Processing Order">
    Operations are applied in a fixed sequence for optimal results:
    1. **Optimization** — reduce file size (max 1MB, max 1920px dimension)
    2. **Resize** — change dimensions (px or % units, aspect ratio preserved)
    3. **Crop** — extract specific regions with position control
    4. **Format Conversion** — convert to JPG, PNG, WEBP, or GIF
  </Step>

  <Step title="Memory Management">
    Intelligent memory handling prevents browser crashes:
    - Files over 10MB get low-resolution previews (300px)
    - Memory usage estimation warns users before issues arise
    - URL cleanup with `revokeObjectURL` prevents memory leaks
  </Step>
</Steps>

## How It Works

PixelFlow's architecture is designed for performance and reliability:

<Note>
  **Processing Strategy**: Images are processed in batches of 3 to prevent memory issues. Each batch completes before the next begins, with a 10ms delay for browser rendering updates.
</Note>

### Component Structure

```
app/                      # Next.js App Router
  layout.jsx              # Root layout with metadata
  [[...slug]]/
    page.jsx              # Catch-all route for static export
    client.jsx            # Client wrapper with BrandProvider

src/                      # Application source
  App.jsx                 # Main app component
  components/
    ImageUploader/        # UploadArea, UploadedImagesList
    ImageProcessor/       # ProcessedImagesList
    Tools/                # ToolsPanel, ResizeTool, CropTool
    RenameTools/          # RenamePanel, SequentialRename
    Common/               # BrandLogo, BrandConfigPanel
  hooks/                  # useImageUpload, useImageProcessor, useImageRename
  lib/                    # imageProcessing, memoryManager, logger, constants
  context/                # BrandContext for theme/branding
  config/                 # brandConfig defaults
```

### Context & State

<Tip>
  **BrandContext** manages application branding (logo, colors, app name) with localStorage persistence under the key `pixelflow_brand_settings`.
</Tip>

The application uses a single-view architecture with no routing library. State flows through three main hooks:

```jsx src/App.jsx
function App() {
  const { uploadedImages, handleUpload, removeImage } = useImageUpload();
  
  const {
    processedImages,
    processImages,
    toggleOptimize,
    changeFormat,
    updateResizeSettings,
    updateCropSettings,
  } = useImageProcessor(uploadedImages);
  
  const {
    renamedImages,
    toggleCleanText,
    changeRenameMode,
    updateSequentialSettings,
  } = useImageRename(processedImages);
  
  // UI renders uploadedImages, processedImages, and renamedImages
}
```
