# Quick Start Guide

Get PixelFlow running locally in just a few minutes. This guide walks you through installation, first run, and processing your first batch of images.

<Note>
  **Prerequisites**: You'll need Node.js 18+ and npm installed on your system. Check with `node --version` and `npm --version`.
</Note>

---

## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/jios325/pixelflow.git
    cd pixelflow
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
    
    This installs all required packages:
    - React 19 and Next.js 16
    - Ant Design UI components
    - Image processing libraries (`browser-image-compression`, `react-image-file-resizer`)
    - File handling utilities (`jszip`, `file-saver`)
  </Step>

  <Step title="Start Development Server">
    ```bash
    npm run dev
    ```
    
    The app will start on **http://localhost:3000** with Turbopack for fast hot-reloading.
  </Step>

  <Step title="Open in Browser">
    Navigate to [http://localhost:3000](http://localhost:3000) to see PixelFlow in action.
  </Step>
</Steps>

<Check>
  **Success!** You should see the PixelFlow upload interface with a violet-themed design.
</Check>

## Your First Image Processing Workflow

Let's process a batch of images with optimization, resizing, and format conversion.

### Step 1: Upload Images

<Tabs>
  <Tab title="Drag & Drop">
    1. Drag images from your file manager
    2. Drop them onto the upload area
    3. Images appear in the **Imágenes subidas** panel
    
    <Tip>
      You can upload up to **20 images** at once. Files over 60MB will be rejected.
    </Tip>
  </Tab>
  
  <Tab title="File Picker">
    1. Click the upload area
    2. Select images from the file dialog
    3. Click **Open** to upload
    
    Supported formats: JPG, PNG, WEBP, GIF, and other image formats.
  </Tab>
</Tabs>

### Step 2: Configure Processing Tools

Once images are uploaded, the **Tools Panel** appears on the left.

<AccordionGroup>
  <Accordion title="1. Enable Optimization">
    The **Optimize** toggle is enabled by default. This compresses images to:
    - Maximum size: **1 MB**
    - Maximum dimension: **1920px**
    
    ```jsx
    // Configuration in src/lib/constants.js
    export const OPTIMIZE_MAX_SIZE_MB = 1;
    export const OPTIMIZE_MAX_DIMENSION = 1920;
    ```
    
    <Note>
      Leave this enabled for most use cases. It significantly reduces file sizes without visible quality loss.
    </Note>
  </Accordion>

  <Accordion title="2. Choose Output Format">
    In the **Formato de salida** section:
    
    <Steps>
      <Step title="Open the format dropdown">
        Options: Original, JPG, PNG, WEBP, GIF
      </Step>
      
      <Step title="Select a format">
        For this example, choose **WEBP** for best compression.
      </Step>
    </Steps>
    
    <Tip>
      **WEBP** offers 25-35% better compression than JPG while maintaining quality.
    </Tip>
  </Accordion>

  <Accordion title="3. Set Resize Dimensions">
    In the **Redimensión** section:
    
    1. Enable resizing by setting width/height values
    2. Choose unit: **Píxeles** or **Porcentaje**
    3. Enable **Mantener proporción** to preserve aspect ratio
    
    **Example: Resize to 1200px width**
    ```
    Width: 1200
    Height: (auto-calculated based on aspect ratio)
    Unit: Píxeles
    Mantener proporción: ✓
    ```
    
    The height will automatically adjust. For a 16:9 image:
    ```
    1200 ÷ 16 × 9 = 675px height
    ```
  </Accordion>

  <Accordion title="4. Configure Cropping (Optional)">
    In the **Corte de imagen** section:
    
    1. Select crop type:
       - **Cuadrado (1:1)** — 1000x1000
       - **Horizontal (16:9)** — 1920x1080
       - **Horizontal (4:3)** — 1600x1200
       - **Vertical (9:16)** — 1080x1920
       - **Vertical (3:4)** — 1200x1600
    
    2. Choose position: Center, Top Left, Top Right, Bottom Center, etc.
    
    <Warning>
      Cropping is applied **after resizing**. If you resize to 1200px width, the crop dimensions must fit within that size.
    </Warning>
  </Accordion>
</AccordionGroup>

### Step 3: Process Images

Once you've configured your tools:

<Steps>
  <Step title="Click the Optimizar button">
    Located in the **Imágenes Listas** card header (top-right).
  </Step>
  
  <Step title="Watch the processing">
    Images are processed in batches of 3. You'll see:
    - Progress indicator for large batches
    - Processed images appearing in real-time
  </Step>
  
  <Step title="Review results">
    Each processed image shows:
    - Thumbnail preview
    - File name
    - File size and format
    - Size reduction percentage
  </Step>
</Steps>

### Step 4: Rename Files (Optional)

The **Renombrado de archivos** panel provides three renaming modes:

<Tabs>
  <Tab title="Sequential">
    Create numbered filenames:
    
    ```
    Prefix: photo_
    Start Number: 1
    Digits: 3
    ```
    
    **Output:**
    ```
    photo_001.webp
    photo_002.webp
    photo_003.webp
    ```
  </Tab>
  
  <Tab title="Add Text">
    Add prefix or suffix to existing names:
    
    ```
    Text: vacation_
    Position: Prefix
    ```
    
    **Output:**
    ```
    vacation_IMG_1234.webp
    vacation_IMG_1235.webp
    ```
  </Tab>
  
  <Tab title="Replace Text">
    Find and replace in filenames:
    
    ```
    Search: IMG
    Replace: photo
    ```
    
    **Output:**
    ```
    IMG_1234.webp → photo_1234.webp
    ```
  </Tab>
</Tabs>

<Tip>
  Enable **Limpiar nombres** to automatically remove special characters and convert to lowercase with underscores.
</Tip>

### Step 5: Download Results

You have two download options:

<CardGroup cols={2}>
  <Card title="Individual Download" icon="download">
    Hover over any processed image and click the download icon to save that file.
  </Card>
  
  <Card title="Batch Download" icon="file-zipper">
    Click the **Download All** button to get a ZIP archive with all processed images.
  </Card>
</CardGroup>

<Note>
  Batch downloads are created in groups of 5 images at a time using `jszip` and `file-saver`.
</Note>

## Understanding the Processing Pipeline

PixelFlow processes images in a specific order for optimal results:

```mermaid
graph LR
    A[Upload] --> B[1. Optimize]
    B --> C[2. Resize]
    C --> D[3. Crop]
    D --> E[4. Convert Format]
    E --> F[5. Rename]
    F --> G[Download]
```

<Steps>
  <Step title="1. Optimization">
    Compress to max 1MB and 1920px using `browser-image-compression`
  </Step>
  
  <Step title="2. Resizing">
    Change dimensions with `react-image-file-resizer` (if enabled)
  </Step>
  
  <Step title="3. Cropping">
    Extract region using custom canvas-based cropping (if enabled)
  </Step>
  
  <Step title="4. Format Conversion">
    Convert to target format via canvas `toBlob()` (if not "original")
  </Step>
  
  <Step title="5. Renaming">
    Apply rename rules reactively via `useEffect` hook
  </Step>
</Steps>

### Code Example

```jsx src/hooks/useImageProcessor.js
const processImages = async () => {
  const batch = uploadedImages.slice(0, 3); // Process 3 at a time
  
  const results = await Promise.all(
    batch.map(async (img) => {
      let processedFile = img.file;
      
      // 1. Optimize
      if (processingSettings.optimize) {
        processedFile = await optimizeImage(processedFile, {
          maxSizeMB: OPTIMIZE_MAX_SIZE_MB,
          maxWidthOrHeight: OPTIMIZE_MAX_DIMENSION,
          useWebWorker: true,
        });
      }
      
      // 2. Resize
      if (processingSettings.resize.enabled) {
        const { width, height } = calculateDimensions();
        processedFile = await resizeImage(
          processedFile,
          width,
          height,
          processingSettings.resize.maintainAspectRatio
        );
      }
      
      // 3. Crop
      if (processingSettings.crop.enabled) {
        processedFile = await cropImage(
          processedFile,
          processingSettings.crop.width,
          processingSettings.crop.height,
          processingSettings.crop.position
        );
      }
      
      // 4. Convert Format
      if (processingSettings.format !== 'original') {
        processedFile = await convertImageFormat(
          processedFile,
          processingSettings.format
        );
      }
      
      return { ...img, processedFile, processed: true };
    })
  );
  
  setProcessedImages(results);
};
```

## Common Use Cases

<AccordionGroup>
  <Accordion title="Web Optimization (Blog/Website Images)">
    **Goal**: Reduce file sizes for fast page loads
    
    **Settings**:
    - ✓ Optimize: Enabled
    - Format: **WEBP**
    - Resize: **1920px** width (max)
    - Crop: Disabled
    
    **Result**: Images optimized for web with 50-70% size reduction
  </Accordion>

  <Accordion title="Social Media (Instagram Posts)">
    **Goal**: Create square images for Instagram feed
    
    **Settings**:
    - ✓ Optimize: Enabled
    - Format: **JPG**
    - Resize: **1080px** (will become 1080x1080 after crop)
    - Crop: **Cuadrado (1:1)**, Position: Center
    
    **Result**: Perfect 1080x1080px Instagram images
  </Accordion>

  <Accordion title="Email Attachments (Reduce Size)">
    **Goal**: Shrink images to fit email size limits
    
    **Settings**:
    - ✓ Optimize: Enabled
    - Format: **JPG**
    - Resize: **800px** width
    - Crop: Disabled
    
    **Result**: Small files (typically &lt;200KB) suitable for email
  </Accordion>

  <Accordion title="Product Photography (Consistent Sizing)">
    **Goal**: Create uniform product images with white background
    
    **Settings**:
    - ✓ Optimize: Enabled
    - Format: **JPG** (adds white background)
    - Resize: **1200px** × **1200px**
    - Crop: **Cuadrado (1:1)**, Position: Center
    - Rename: **Sequential** (product_001, product_002, ...)
    
    **Result**: Consistent, professional product images
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Images not uploading">
    **Check**:
    - File size < 60MB
    - Valid image format (JPG, PNG, WEBP, GIF)
    - Browser console for errors (`F12` → Console tab)
    
    **Solution**: Try smaller files or check file permissions.
  </Accordion>

  <Accordion title="Processing is slow">
    **Causes**:
    - Large image files (>10MB)
    - Many images (>10 at once)
    - Low system memory
    
    **Solution**:
    - Process in smaller batches
    - Close other browser tabs
    - Use optimization to reduce file sizes first
  </Accordion>

  <Accordion title="Browser freezes during processing">
    **Fix**:
    - Refresh the page
    - Reduce batch size (upload fewer images)
    - Check memory usage in Task Manager
    
    **Note**: Batching is set to 3 images at a time in `src/lib/constants.js`. You can reduce this if needed:
    
    ```js
    export const PROCESSING_BATCH_SIZE = 2; // Reduce from 3 to 2
    ```
  </Accordion>

  <Accordion title="Downloaded ZIP is empty">
    **Causes**:
    - Images not fully processed yet
    - Browser blocking downloads
    
    **Solution**:
    - Wait for processing to complete (all images show in "Imágenes Listas")
    - Check browser download settings/permissions
    - Try downloading individual images first
  </Accordion>
</AccordionGroup>

## Build for Production

To deploy PixelFlow as a static site:

<Steps>
  <Step title="Build the app">
    ```bash
    npm run build
    ```
    
    This creates a static export in the `build/` directory using Next.js static export.
  </Step>
  
  <Step title="Test the build">
    ```bash
    npm start
    ```
    
    Preview the production build locally on http://localhost:3000
  </Step>
  
  <Step title="Deploy">
    Upload the `build/` folder to any static host:
    - **Vercel**: `vercel deploy`
    - **Netlify**: Drag & drop the `build/` folder
    - **GitHub Pages**: Push to `gh-pages` branch
    - **AWS S3**: Upload to S3 bucket with static website hosting
  </Step>
</Steps>

<Tip>
  The built app is a fully static SPA with no server dependencies. It works on any static file host.
</Tip>
